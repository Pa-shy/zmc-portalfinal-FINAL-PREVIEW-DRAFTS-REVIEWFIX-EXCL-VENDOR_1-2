SECTION 1 — CORE WORKFLOWS TO ENFORCE
────────────────────────────────────────────────────────

A) MEDIA PRACTITIONER NEW ACCREDITATION (STANDARD FLOW)
FLOW:
Applicant → Accreditation Officer → (Registrar + Accounts) → Accounts Verification → Production (Accreditation Officer)

1) Applicant submits new accreditation
- Status: SubmittedToAccreditationOfficer
- Only Accreditation Officer sees it first.

2) Accreditation Officer dashboard actions
- Review fields + documents
- Assign category
- If missing/wrong: “Return to Applicant” with message (mandatory reason)
  Status: ReturnedToApplicant
- If OK: “Approve”
  Status: ApprovedByAccreditationOfficer_AwaitingPayment
  After approval:
   - Application becomes visible to Registrar AND Accounts dashboards (read queues)
   - Applicant is immediately prompted to pay.

3) Payment prompt after Accreditation Officer approval
Option 1: PayNow
- “Pay with PayNow” opens PayNow UI
- After payment: applicant clicks “Done”
- Show modal requiring PayNow reference number
- Submit reference → Status: AwaitingAccountsVerification

Option 2: Upload Payment Proof/Related Doc
- Modal for proof upload + optional payment metadata
- Submit → Status: AwaitingAccountsVerification

4) Registrar dashboard
- Receives only ApprovedByAccreditationOfficer_AwaitingPayment applications
- Registrar reviews documents vs form data
- If errors: Registrar sends “Fix Request” to Accreditation Officer (internal message/task)
  Status: RegistrarRaisedFixRequest
  Accreditation Officer fixes and re-approves/confirm changes.
- Registrar does NOT verify payments.

5) Accounts dashboard
- Receives AwaitingAccountsVerification applications
- PayNow: compare submitted reference to external PayNow platform record
  - Match → approve → Status: PaymentVerified
  - No match → reject with reason → Status: PaymentRejected (applicant must resubmit)
- Proof: verify against internal records → approve/reject similarly.
- PaymentVerified applications are routed to Production.

6) Production dashboard (Accreditation Officer only)
- Shows only PaymentVerified applications
- Generate + print card/certificate
- Track: template version, who generated, who printed, print count, timestamps
- Status: ProducedReadyForCollection
- Add a sidebar link “Production” in Accreditation Officer menu for easy access.

B) SPECIAL/COMPLICATED PAYMENTS (WAIVERS)
Requirement:
Some applications use waivers as payment proof. Accreditation Officer must be able to forward to Registrar WITHOUT approving.

Add Accreditation Officer action:
- “Forward to Registrar (No Approval)”
  Status: ForwardedToRegistrar_NoAccredApproval
  Must capture reason (e.g., waiver/special case)

Registrar then:
- Reviews and if satisfied:
  - Pushes to Accounts from Registrar dashboard
  Status: PendingAccountsReview_FromRegistrar

Accounts:
- Verifies waiver document as payment-equivalent
- Approves → PaymentVerified → Production
- Rejects → PaymentRejected with reason

C) REGISTRAR OVERSIGHT OF ACCOUNTS
Registrar must have READ-ONLY oversight into Accounts activity:
- View payments queue and payment outcomes
- View paynow references, proofs, waivers, receipt records
- View payment audit logs and counts
Registrar cannot approve payments.

D) MEDIA HOUSE NEW REGISTRATION (TWO-STAGE PAYMENT FLOW)
FLOW:
Applicant → (Application Fee Payment) → Accreditation Officer → Registrar → (Registration Fee Payment) → Accounts → Production

Stage 1: Submission requires Application Fee
On submitting Media House registration:
- Prompt for Application Fee:
  Option 1: PayNow → Done → enter reference → attach to application
  Option 2: Upload application fee proof via modal (details + file)
- Block submission if application fee was not submitted.
- Status: SubmittedWithApplicationFee_ToAccreditationOfficer

Stage 2: Accreditation Officer review
- Verify docs/fields/requirements met
- If incomplete: return to applicant with message
- If complete: verify/push to Registrar
  Status: VerifiedByAccreditationOfficer_PendingRegistrarReview

Stage 3: Registrar review
- If errors: Fix Request to Accreditation Officer
- If satisfied: Registrar approval requires uploading OFFICIAL LETTER (mandatory)
  - Block approve if letter missing
- After approval:
  Status: RegistrarApproved_PendingRegistrationFeePayment
  Applicant is prompted to pay registration fee.

Stage 4: Registration fee payment (after registrar approval)
- Option 1: PayNow → Done → enter reference
- Option 2: Upload proof
- Status: AwaitingAccountsVerification

Accounts verifies registration fee
- Approve → PaymentVerified → Production
- Reject → PaymentRejected

Production by Accreditation Officer.

  Applicant → Accounts (payment approval) → Production (Accreditation Officer)

RENEWAL DOCUMENT RULES:

MEDIA HOUSE RENEWAL REQUIRED DOCS:

- Official Letter requesting renewal (required)
